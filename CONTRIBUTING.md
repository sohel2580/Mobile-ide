# Contributing to Kora AI 🚀

First off, thank you for considering contributing to Kora AI! It's people like you that make Kora AI such a great tool for the community.

We welcome all contributions: bug fixes, feature implementations, documentation improvements, and more!

## 🌟 Why Contribute?
Kora AI aims to be the **most advanced open-source AI code editor in the browser**. By contributing:
- You help build a tool that thousands of developers use.
- You get to work with modern technologies like Next.js, Monaco Editor, Hugging Face AI, and complex UI states.
- You build your open-source portfolio and get recognized as a core contributor!

## 🛠️ How to Contribute

### 1. Fork and Clone
1. Fork the repository on GitHub.
2. Clone your fork locally:
   \`\`\`bash
   git clone https://github.com/yourusername/kora-ai-chat.git
   \`\`\`

### 2. Branching
Create a new branch for your feature or bug fix:
\`\`\`bash
git checkout -b feature/your-awesome-feature
# or
git checkout -b fix/bug-description
\`\`\`

### 3. Development
- Install dependencies: \`npm install\`
- Start the server: \`npm run dev\`
- Make your changes. Ensure the UI looks premium and works flawlessly.

### 4. Commit Guidelines
We use a standard commit message format:
- \`feat:\` A new feature
- \`fix:\` A bug fix
- \`docs:\` Documentation changes
- \`style:\` UI or styling changes (CSS, Tailwind)
- \`refactor:\` Code changes that neither fix a bug nor add a feature

Example: \`feat: added multi-tab support in the editor\`

### 5. Submit a Pull Request
- Push your changes to your fork.
- Open a Pull Request (PR) against our \`main\` branch.
- Fill out the PR template carefully. Include screenshots or videos if you changed the UI.

## ⚖️ Code of Conduct
Please note that this project is released with a Contributor Code of Conduct. By participating in this project you agree to abide by its terms. Be respectful and collaborative!
